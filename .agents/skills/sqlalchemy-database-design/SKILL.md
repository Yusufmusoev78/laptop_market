# SQLAlchemy Database Design Skill

This skill governs the creation and modification of database schemas in this project.

## Core Guidelines

1.  **SQLAlchemy 2.0 Style**:
    - Use the new 2.0 style syntax (e.g., `Mapped` and `mapped_column`).
    - Example:
      ```python
      from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column
      from sqlalchemy import String, DateTime, func
      from datetime import datetime

      class Base(DeclarativeBase):
          pass

      class User(Base):
          __tablename__ = "users"
          
          id: Mapped[int] = mapped_column(primary_key=True)
          email: Mapped[str] = mapped_column(String(255), unique=True, index=True)
          hashed_password: Mapped[str] = mapped_column(String(255))
          created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
          updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now(), onupdate=func.now())
      ```

2.  **Clean Schema Design**:
    - Use meaningful, descriptive table and column names (snake_case).
    - Always define Primary Keys, Foreign Keys, and necessary Indexes to optimize query performance.
    - Keep audit fields like `created_at` and `updated_at` on mostly all tables.
    - Prefer UUIDs or BigInts for primary keys on large tables or when distributing data.
    - Prefer ENUM types or simple lookup tables for strict categorizations.

3.  **Migrations (Alembic)**:
    - Never modify the database schema directly in the database.
    - Always generate Alembic migrations for any model changes: `alembic revision --autogenerate -m "description"`.
    - Always review autogenerated migration files before applying them. Autogeneration is not perfect, especially for constraints and renames.
    - Ensure your `env.py` has the target metadata correctly configured (`target_metadata = Base.metadata`).

4.  **Async Support (If using Async Drivers)**:
    - If configured to use async databases, use `AsyncSession` and `async_sessionmaker` for database interactions to maintain high concurrency in FastAPI.
    - Remember that lazy loading does not work seamlessly with async sessions without specific configurations or strategies (e.g., `selectinload`, `joinedload`). Explicitly eager load relationships when needed.
